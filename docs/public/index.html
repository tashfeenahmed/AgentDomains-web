AgentDNS

AgentDNS gives any AI agent (or human) a free, real subdomain under makes.fyi — claimable from a single CLI command. Point it at an IP, a CNAME, delegate it to your own nameservers, or add TXT records for SSL.

API base: https://api.makes.fyi · CLI: github.com/tashfeenahmed/AgentDNS · source-available (FSL-1.1).

Install

# with the Go toolchain
go install github.com/tashfeenahmed/AgentDNS/cmd/agentdns@latest

# verify
agentdns --help

Prebuilt binaries for macOS/Linux/Windows are attached to GitHub Releases.

Quickstart

agentdns signup                                   # instant account + API key
agentdns claim myagent --type A --content 203.0.113.10
dig +short myagent.makes.fyi                       # 203.0.113.10 ✨
No email is required to start. Your account is provisional for 30 days — validate it (below) to keep it and raise your quota.

Accounts & validation

Signup creates a provisional account with a quota of 1 subdomain and a 30-day clock. To make it permanent and raise the quota to 3, a human attaches and confirms an email:

agentdns email you@example.com   # we send a link; a human clicks it
agentdns whoami                  # state: verified once confirmed

If an account is never validated within 30 days, it and its subdomains are automatically removed and the names are released.

DNS records

Claim a label, optionally with a record, then add more any time:

agentdns claim shop --type CNAME --content myshop.vercel.app
agentdns record shop --type A --content 203.0.113.10 --host www
agentdns get shop                 # show all records
agentdns get shop --json          # machine-readable
TypeUse
A / AAAAPoint at an IPv4 / IPv6 address.
CNAMEAlias to another hostname (e.g. a PaaS).
TXTVerification / ACME challenges (see SSL below).

Getting HTTPS / SSL

AgentDNS is a DNS provider — you bring your own certificate. Two easy paths:

HTTP-01 (simplest)

Point your subdomain at your server, then let Let's Encrypt verify over HTTP:

agentdns claim mybot --type A --content "$(curl -s ifconfig.me)"
sudo certbot certonly --standalone -d mybot.makes.fyi

DNS-01 (no inbound server needed)

Add the ACME challenge token as a TXT record:

agentdns txt mybot "<token-from-acme-client>" --host _acme-challenge

Nameserver delegation

Want full control of the subtree (your own records, MX, sub-subdomains)? Delegate it to your nameservers:

agentdns ns mybot ns1.yourdns.com ns2.yourdns.com

From then on, mybot.makes.fyi and everything under it is served by your nameservers.

Using it from agents

Every command accepts --json for clean machine output, and reads credentials from the environment so no interactive setup is needed:

export AGENTDNS_API_KEY=adns_...           # skip local config
agentdns claim $NAME --type A --content $IP --json | jq -r .fqdn

You can also call the HTTP API directly (below) — handy inside sandboxes.

CLI commands

CommandDescription
agentdns signupCreate an account; saves the key to ~/.agentdns/config.json
agentdns whoamiAccount state, quota, usage
agentdns email <addr>Attach an email for human validation
agentdns claim <label>Claim <label>.makes.fyi (+ optional --type/--content/--host)
agentdns listList your subdomains
agentdns get <label>Show one subdomain and its records
agentdns record <label>Add a record: --type --content [--host]
agentdns ns <label> <ns>…Delegate to your nameservers
agentdns txt <label> <value>Add a TXT record (--host _acme-challenge)
agentdns delete <label>Delete a subdomain and its records

Global flags: --json, --api-url. Env: AGENTDNS_API_KEY, AGENTDNS_API_URL.

HTTP API

Base URL https://api.makes.fyi. Authenticate with Authorization: Bearer <api_key>.

Method & pathDescription
POST /v1/signupCreate a provisional account, returns api_key
GET /v1/whoamiAccount info
POST /v1/account/email{"email":"…"} — send validation link
POST /v1/subdomains{"label","type?","content?","host?"}
GET /v1/subdomainsList
GET /v1/subdomains/{label}Show one
POST /v1/subdomains/{label}/recordsAdd a record
PUT /v1/subdomains/{label}/ns{"nameservers":[…]}
DELETE /v1/subdomains/{label}Delete
# example: claim with curl
KEY=$(curl -s -X POST https://api.makes.fyi/v1/signup | jq -r .api_key)
curl -s -X POST https://api.makes.fyi/v1/subdomains \
  -H "Authorization: Bearer $KEY" \
  -d '{"label":"myagent","type":"A","content":"203.0.113.10"}'
