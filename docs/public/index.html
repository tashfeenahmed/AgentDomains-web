AgentDomains

When an AI agent builds a website or an API, it needs a domain to serve it on. AgentDomains hands out free ones under makes.fyi (the default) and agentdomains.co, claimable from a single CLI command. Point a domain at an IP or a CNAME, delegate it to your own nameservers, or add TXT records to get HTTPS.

API base: https://api.agentdomains.co · CLI: github.com/tashfeenahmed/AgentDomains · source-available (FSL-1.1).

Install

# with the Go toolchain
go install github.com/tashfeenahmed/AgentDomains/cmd/agentdomains@latest

# verify
agentdomains --help

Prebuilt binaries for macOS/Linux/Windows are attached to GitHub Releases.

Quickstart

agentdomains signup                                   # instant account + API key
agentdomains claim myagent --type A --content 203.0.113.10
dig +short myagent.makes.fyi                       # 203.0.113.10 ✨
No email is required to start. Your account stays provisional for 30 days. Validate it (below) to keep it and raise your quota.

Choosing a domain

Names are claimed under makes.fyi by default. Pass --domain agentdomains.co (CLI) or a "domain" field (API) to claim under the other one. The same label can exist under each domain independently; scope get/delete with --domain when a label is claimed under both.

agentdomains claim myagent --domain agentdomains.co   # myagent.agentdomains.co

Accounts & validation

Signup creates a provisional account with a quota of 1 domain and a 30-day clock. To make it permanent and raise the quota to 3, a human attaches and confirms an email:

agentdomains email you@example.com   # we send a link; a human clicks it
agentdomains whoami                  # state: verified once confirmed

If an account is never validated within 30 days, it and its domains are automatically removed and the names are released.

DNS records

Claim a label, optionally with a record, then add more any time:

agentdomains claim shop --type CNAME --content myshop.vercel.app
agentdomains record shop --type A --content 203.0.113.10 --host www
agentdomains get shop                 # show all records
agentdomains get shop --json          # machine-readable
TypeUse
A / AAAAPoint at an IPv4 / IPv6 address.
CNAMEAlias to another hostname (e.g. a PaaS).
TXTVerification / ACME challenges (see SSL below).

Getting HTTPS / SSL

AgentDomains handles DNS, so you bring your own certificate. Two paths:

HTTP-01 (simplest)

Point your domain at your server, then let Let's Encrypt verify over HTTP:

agentdomains claim mybot --type A --content "$(curl -s ifconfig.me)"
sudo certbot certonly --standalone -d mybot.makes.fyi

DNS-01 (no inbound server needed)

Add the ACME challenge token as a TXT record:

agentdomains txt mybot "<token-from-acme-client>" --host _acme-challenge

URL forwarding

Forward a name to any URL with a real HTTP redirect, served at the edge with valid HTTPS. forward claims the label first if you don't own it yet, so it's a single step:

agentdomains forward me https://my-portfolio.example.com
# me.makes.fyi -> 302 redirect to https://my-portfolio.example.com

agentdomains forward me https://dest.com --permanent        # 301 instead of 302
agentdomains forward me https://dest.com --no-preserve-path # land on target root
agentdomains unforward me                                   # remove the forward

The request path and query are preserved by default (me.makes.fyi/blog?x=1 → …/blog?x=1). A forward replaces address records: a label can't hold both a forward and an A/AAAA/CNAME at once (TXT records still coexist).

Forwarding is an HTTP redirect, not hosting. The visitor's browser ends up on the target URL. To serve your own content on the name instead, point an A/CNAME record at your server.

Nameserver delegation

Want full control of the subtree, with your own records, MX, and sub-domains? Delegate it to your nameservers:

agentdomains ns mybot ns1.yourdns.com ns2.yourdns.com

From then on, mybot.makes.fyi and everything under it is served by your nameservers.

Using it from agents

Every command accepts --json for clean machine output, and reads credentials from the environment so no interactive setup is needed:

export AGENTDOMAINS_API_KEY=adom_...           # skip local config
agentdomains claim $NAME --type A --content $IP --json | jq -r .fqdn

You can also call the HTTP API directly (below), which is handy inside sandboxes.

CLI commands

CommandDescription
agentdomains signupCreate an account; saves the key to ~/.agentdomains/config.json
agentdomains whoamiAccount state, quota, usage
agentdomains email <addr>Attach an email for human validation
agentdomains claim <label>Claim <label>.makes.fyi (or --domain agentdomains.co; + optional --type/--content/--host)
agentdomains listList your domains
agentdomains get <label>Show one domain and its records
agentdomains record <label>Add a record: --type --content [--host]
agentdomains forward <label> <url>Forward to a URL (--permanent, --no-preserve-path); claims the label if needed
agentdomains unforward <label>Remove a forward
agentdomains ns <label> <ns>…Delegate to your nameservers
agentdomains txt <label> <value>Add a TXT record (--host _acme-challenge)
agentdomains delete <label>Delete a domain and its records

Global flags: --json, --api-url, --domain. Env: AGENTDOMAINS_API_KEY, AGENTDOMAINS_API_URL.

HTTP API

Base URL https://api.agentdomains.co. Authenticate with Authorization: Bearer <api_key>.

Method & pathDescription
POST /v1/signupCreate a provisional account, returns api_key
GET /v1/whoamiAccount info
POST /v1/account/email{"email":"…"}, sends a validation link
POST /v1/subdomains{"label","domain?","type?","content?","host?"}
GET /v1/subdomainsList
GET /v1/subdomains/{label}Show one
POST /v1/subdomains/{label}/recordsAdd a record
PUT /v1/subdomains/{label}/ns{"nameservers":[…]}
PUT /v1/subdomains/{label}/forward{"target","permanent?","preserve_path?","cloak?"}
DELETE /v1/subdomains/{label}/forwardRemove a forward
DELETE /v1/subdomains/{label}Delete
# example: claim with curl
KEY=$(curl -s -X POST https://api.agentdomains.co/v1/signup | jq -r .api_key)
curl -s -X POST https://api.agentdomains.co/v1/subdomains \
  -H "Authorization: Bearer $KEY" \
  -d '{"label":"myagent","type":"A","content":"203.0.113.10"}'
